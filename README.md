# AdeyApp Monorepo

This repository now contains the planning documents and the initial monorepo scaffold for the multi-tenant spa management platform.

Current workspace layout:

- `apps/` for deployable client applications
- `packages/` for shared code
- `services/` for backend runtime services
- `prisma/` for schema and migration work
- `docs/` for repo-level documentation helpers

Planning documents currently live at the repo root and drive the implementation sequence.
